Implement a generic game module framework in packages/domain.

Requirements:

  1. Create a generic GameRules interface.

It must support: - gameType - version - getInitialState(config) - getLegalMoves(state) - validateMove(state, move) - applyMove(state, move) - getResult(state) - serializeState(state) - hashState(state)

  1. Create shared domain types:
  2. GameType
  3. GameMode
  4. GameResult
  5. ValidationResult
  6. PlayerSymbol
  7. GameConfig
  8. SerializedGameState

  9. StateHash

  10. Wrap existing deterministic tic-tac-toe and super tic-tac-toe logic behind this interface.

  11. Create a gameRegistry that resolves game rules by gameType.

  12. Ensure the registry can support future game modules without changing API route logic.

  13. Domain rules must be pure:

  14. no database
  15. no API calls
  16. no wallet references
  17. no settlement references
  18. no rank/reward/progression references
  19. no timestamps inside state transition functions

  20. no random behavior unless deterministic seed support is explicitly added

  21. Add tests for:

  22. initial state generation
  23. legal move generation
  24. invalid move rejection
  25. deterministic applyMove behavior
  26. result detection
  27. serialization
  28. state hashing
  29. replayed move sequence produces expected final hash
