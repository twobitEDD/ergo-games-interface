Release Plan

This document tracks acceptance criteria and readiness checks per release increment.

Planned Sequence

  1. Repository bootstrap and baseline architecture rails.
  2. Release 1: Ergo-native wallet-aware gameplay slice.
  3. Release 2: Unfunded participation with off-chain rewards.
  4. Release 3: Gasless server-sponsored settlement path.

Gate Checklist Template

  • Scope and acceptance criteria defined.
  • Core domain rules validated.
  • API/UI behavior tested.
  • Security/compliance notes updated.
  • Operational readiness verified.

Gate References

  • docs/ekb-integration.md: required EKB-backed workflow and contract audit checks.
  • docs/compliance-gates.md: no-wager boundaries, prohibited language, and release go/no-go controls.
